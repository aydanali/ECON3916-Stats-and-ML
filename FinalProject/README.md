## Overview

This project investigates the quantitative relationship between greenhouse gas emissions, energy consumption, and GDP across 201 countries from 1990 to 2024. Using the Our World in Data CO₂ dataset, three regression models — OLS, Ridge, and Random Forest — were trained to predict log(GDP) from a curated set of emissions and energy features. The analysis was built end-to-end in Python and deployed as an interactive Streamlit dashboard for use by policymakers, environmental economists, and researchers.

The core finding is unambiguous: emissions and GDP are deeply collinear under the current global economic structure. The Random Forest model explains **95.68% of the variance in log-GDP** (cross-validated R² = 0.9568 ± 0.0046), with `primary_energy_consumption` alone accounting for over 50% of predictive importance. This is not a statistical artifact — it is a structural feature of an economy built on emissions-intensive production.

---

## Objective

> *Can GDP be reliably predicted from emissions and energy data, and what does that relationship imply for the cost of decarbonization?*

Environmental economists and policymakers face a persistent challenge: quantifying the economic consequences of reducing emissions. This project provides a data-driven baseline for that conversation. By establishing the predictive power of emissions features on GDP, it allows stakeholders to reason more precisely about which pollutants carry the highest economic co-dependence, where the marginal cost of reduction is greatest, and what GDP trajectories look like under different emissions scenarios.

The analysis explicitly distinguishes *predictive* relationships from *causal* ones. The models are positioned as forecasting tools, not causal estimators. For policy applications requiring causal identification, the recommendation is to complement this work with fixed-effects panel estimators or instrumental variable strategies.

## Data

**Source:** [Our World in Data CO₂ and Greenhouse Gas Emissions](https://github.com/owid/co2-data)
A collaboration between researchers at the University of Oxford and the Global Change Data Lab. The dataset is the primary emissions reference used by the World Bank, covering all major countries and territories since 1750.

**Raw dataset:** 50,411 observations × 79 features
**Cleaned dataset:** 7,035 observations × 18 features · 201 countries · 1990–2024

### Cleaning Pipeline (`owid_cleaning.py`)

The raw data presented several structural challenges requiring deliberate methodological choices:

| Stage | Action | Rationale |
|---|---|---|
| 1a | Remove 36 regional aggregates (OWID non-country entities) | `iso_code` is null for all aggregates; keeping them causes double-counting |
| 1b | Restrict to 1990–2024 | Pre-1990 GDP and energy coverage is too sparse to interpolate without fabricating data |
| 1c | Drop columns >50% missing | Only the `other_industry` family of columns exceeded this threshold |
| 2 | Drop countries >50% missing on focus columns | Removes 17 micro-territories with no statistical infrastructure; retains all low-income countries |
| 3 | Linear interpolation within each country's time series | Fills ~11% of remaining cells using arithmetic interpolation between observed values — no extrapolation beyond a country's observed range |

**On the missingness mechanism:** The data is best characterized as MNAR (Missing Not at Random), not MAR. Missingness correlates with development level — low-income countries are systematically less likely to report — which means the imputation approach was designed to be conservative: no cross-country imputation, no extrapolation, no fabrication of values where a country simply never reported.

**Outliers:** A Tukey Fences test identified 12,196 outliers across the raw data and 1,137 in the cleaned subset. All were retained. Outliers in this dataset are predominantly high-emission, high-GDP countries — precisely the observations most relevant to the analysis.

### Feature Engineering

All positively-valued, non-normalized features were log-transformed (`log1p`) to address right skew. The final feature set and target:

| Feature | Description | Transform |
|---|---|---|
| `co2_log` | Annual CO₂ emissions (Mt) | log1p |
| `co2_per_unit_energy` | CO₂ intensity of energy supply (kg/kWh) | None |
| `energy_per_capita` | Energy consumption per person (kWh) | None |
| `ghg_per_capita` | Total GHG emissions per person (tCO₂e) | None |
| `methane_log` | Methane emissions (Mt CO₂e) | log1p |
| `nitrous_oxide_log` | Nitrous oxide emissions (Mt CO₂e) | log1p |
| `oil_co2_log` | Oil-combustion CO₂ (Mt) | log1p |
| `primary_energy_consumption_log` | Total primary energy (TWh) | log1p |
| `total_ghg_log` | Total GHG emissions (Mt CO₂e) | log1p |
| **`gdp_log`** | **Target: GDP in USD** | **log1p** |

The correlation matrix showed strong collinearity across all features and between features and GDP (r > 0.85 for most emissions variables). This collinearity is itself a substantive finding — it reflects the degree to which the global economy has been built on emissions-intensive production.

## Methodology

All models were evaluated on an 80/20 train-test split (`random_state=42`). OLS and Ridge models received `StandardScaler`-normalized inputs; Random Forest and Gradient Boosting operated on unscaled features (tree models are invariant to monotonic transformations). Performance was cross-validated with 5-fold CV using R², RMSE, and MAE.

### Models

**OLS Regression** — The linear baseline. Retains all features without coefficient shrinkage, providing the clearest read of the raw linear relationship between emissions and GDP. Susceptible to the high multicollinearity in this feature set, which inflates coefficient variance without biasing predictions meaningfully.

**Ridge Regression** (α = 100, selected via `RidgeCV`) — Regularized linear regression that shrinks correlated coefficients toward zero. Motivated by the strong multicollinearity observed in the correlation matrix. In practice, Ridge produced only marginal gains over OLS (R² 0.5466 vs. 0.5453), indicating that the limiting factor is the linear functional form rather than coefficient instability.

**Random Forest** (200 trees) — An ensemble of decision trees trained on bootstrapped subsamples. Captures non-linear interactions and threshold effects without feature scaling. The dominant model by every metric, achieving R² ≈ 0.95. The performance gap over the linear models reflects the RF's ability to learn the non-linear relationship between energy intensity and economic output at different stages of development.

**Gradient Boosting** (200 estimators, included for robustness) — Introduced to test whether the RF's feature importance concentration was an artifact of the bagging approach. The GB model showed nearly identical feature importance rankings and underperformed RF on all metrics (R² = 0.8510), confirming that `primary_energy_consumption_log` is a genuinely dominant predictor rather than an ensemble-specific quirk.

### Results

| Model | RMSE | MAE | R² |
|---|---|---|---|
| OLS Regression | 1.3356 | 0.9127 | 0.5453 |
| Ridge Regression | 1.3338 | 0.9107 | 0.5466 |
| **Random Forest** | **0.4440** | **0.1881** | **0.9498** |
| Gradient Boosting | 0.7646 | 0.4791 | 0.8510 |

Cross-validated Random Forest R²: **0.9568 ± 0.0046**

### Feature Importance

`primary_energy_consumption_log` accounts for over 50% of the Random Forest's predictive importance. The remaining eight features contribute approximately 10% or less individually. This concentration is consistent with the feature's Pearson correlation of 0.97 with GDP and reflects the degree to which total energy throughput is a near-sufficient statistic for economic output under the current production structure.

> **Caution:** These are *predictive* importances, not causal effects. The collinearity of the feature set means individual importances should not be interpreted as the independent contribution of any one emission source to GDP. For causal identification, fixed-effects panel models with country and year controls are the appropriate framework.

## Streamlit Dashboard

The project includes a production-ready Streamlit dashboard (`app.py`) designed for both expert and non-expert audiences.

### Features

**Tab 1 — Data Overview**
- Summary statistics and KPI cards (countries, year range, observations, features)
- Filterable data table by country and year range
- Interactive global trend area chart (user-selectable metric)
- GDP vs. CO₂ scatter with bubble size encoding energy consumption (log-log scale)
- Full Pearson correlation heatmap

**Tab 2 — Model Explorer**
- Checkbox feature toggles that retrain all three models live on the selected feature subset
- Side-by-side actual vs. predicted scatter panels, residual-coloured on a diverging scale
- Normalised feature importance comparison across OLS, Ridge, and Random Forest
- Residual distribution histogram for all three models

**Tab 3 — GDP Predictor**
- Sidebar sliders in natural units (CO₂ in Mt, not log) — log transforms applied automatically before inference
- Point estimate from the selected model (OLS, Ridge, or RF)
- 90/80/95% prediction interval via **MAPIE conformal prediction** (distribution-free, no normality assumption)
- Prediction interval visualised against the observed log-GDP distribution
- Feature contribution chart (RF only)
- 5-nearest-neighbours lookup: closest historical country-year observations to the user's input

### Performance

- Models and scaler loaded with `@st.cache_resource` — single load per session, ~82 MB peak RAM for RF
- `@st.cache_data` on all data loading and model retraining functions
- Plotly used for all interactive visualizations (hover tooltips, zoom, pan)
- Tested locally on Windows (Anaconda, Python 3.10)

## Applications

**For policymakers:** The dashboard's predictor tab allows users to input a country's emissions profile and receive a GDP estimate with a prediction interval. This enables scenario analysis — for example, estimating the GDP range associated with a 20% reduction in total GHG — with quantified uncertainty.

**For researchers:** The feature importance analysis and model comparison provide a quantitative basis for understanding which emissions variables carry the most economic co-dependence. The explicit separation of predictive and causal interpretation sets up the natural next step: a panel fixed-effects or instrumental variable study to isolate the within-country, within-year effect of emission changes on GDP.

**For economists:** The collinearity structure of the data is itself informative. The near-perfect correlation between `primary_energy_consumption` and GDP (r = 0.97) reflects a global production function that has not yet decoupled energy throughput from economic output. That decoupling — if and when it occurs — will be visible in this kind of model as a structural break in feature importance.

## Limitations and Future Work

**Causal identification:** The models establish strong predictive associations but cannot isolate the causal effect of emissions on GDP. A within-country fixed-effects estimator would control for time-invariant country characteristics (institutions, geography, development path) and provide more defensible policy estimates.

**Feature dominance:** `primary_energy_consumption_log` dominates all tree-based models. This reflects genuine collinearity, not model failure, but it limits the utility of individual feature importances for policy targeting. SHAP values with interaction decomposition would provide more granular attribution.

**Temporal dynamics:** The panel model treats each country-year as an independent observation. A time-series-aware model (e.g., LSTM or panel VAR) could capture momentum effects — the degree to which last year's emissions predict this year's GDP growth.

**Data vintage:** The OWID dataset is updated periodically. Results should be reproduced against the most current release for any live policy application.
